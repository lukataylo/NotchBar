# Installing NotchBar

This project supports two install paths.

## Recommended for Most People

Use the packaged DMG from GitHub Releases.

1. Download [`NotchBar-0.1.0.dmg`](https://github.com/lukataylo/NotchBar/releases/download/v0.1.0/NotchBar-0.1.0.dmg).
2. Open the DMG.
3. Drag `NotchBar.app` into `Applications`.
4. Open `NotchBar` from `Applications`.

Because current builds are unsigned, macOS may warn on first launch.

If that happens:

1. Right-click `NotchBar.app` and choose `Open`.
2. If needed, go to `System Settings -> Privacy & Security -> Open Anyway`.

## From a Source Checkout

If you cloned the repo and want to install from source:

1. Double-click `Install.command`, or
2. Run:

```bash
./install.sh
```

If a prebuilt app exists in `dist/`, the installer uses that first.
If not, it builds the app from source with Swift.

## Building a DMG

To generate a local installer DMG:

1. Double-click `CreateDMG.command`, or
2. Run:

```bash
./scripts/create_dmg.sh
```

That writes:

```text
dist/NotchBar.app
dist/NotchBar-0.1.0.dmg
```

## First-Launch Permissions

NotchBar can monitor sessions without extra permissions, but some workflows need macOS approval:

- Accessibility:
  Required to send input back into Terminal or iTerm.
- Automation:
  Required to control Terminal or iTerm for resume/send-input actions.
- Notifications:
  Optional, used for approval and completion alerts.

## Provider Setup

During onboarding you can choose a default provider and install provider-specific integration:

- Claude Code:
  Installs hook entries in `~/.claude/settings.json`
- Codex:
  Installs a managed `notchbar` profile in `~/.codex/config.toml`

## Troubleshooting

### The app will not open

- Use right-click `Open` the first time.
- Check `System Settings -> Privacy & Security` for an `Open Anyway` prompt.

### Install.command says Swift is missing

- Install Xcode Command Line Tools:

```bash
xcode-select --install
```

- Or skip the source install and use the packaged DMG.

### Claude approvals always auto-approve

Make sure NotchBar is running when Claude hooks fire. Recent builds check for the `NotchBar` process correctly, but older hook scripts generated by renamed builds may need reinstalling from the app menu.

### Resume or send-input does not work

Grant Accessibility and Automation permissions for NotchBar in macOS privacy settings.
